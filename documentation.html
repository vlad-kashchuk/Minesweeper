Minesweeper documentation

To be able to load JSON lvl you will need to set up the local server (shoud be set to the root folder). The file's path is model/gameState.json. By default the page loads the beginner lvl.
The timer is set(starts) after the first cell is uncovered and stops if you hit a mine(lose) or open all the cells not containing the mines. Setting flags is voluntarily, all it does - sets the flagged cell inactive, so you can't click on it accidentally, the cell gets back active if the flag is removed. There is a mine counter in the upper right corner of the grid indicating the total number of the mines, the number is decremented when the flag is set. The number can go into negative, indicating that the player placed more flags than there mines on the grid. The game grid is drawn by JS function drawTable() in model.js, the function is scalable and can draw a grid of any size. Function createGame() in model generates a random(different every time) layout by calling it's prototype methods. Each time the player looses or wins the game, the player's score is stored local storage and being dispayed next time you load the page.

There are 4 different audio events implemented in the game:

  • click - click sound is played when the left button is clicked over an active cell.
  • flag - flag sound is played when a flag is set in a cell(not played when removed).
  • explosion - explosion sound is played when the player hits a mine.
  • you won - you won sound is played when the game is won.

The game has 4 different levels of difficulty in the game. A player can load any of the 4 levels by chosing an option in the select menu, the level is loaded automatically.

  • Beginner (grid: 9x9, 10 mines)
  • Intermediate (grid: 16x16, 40 mines)
  • Expert (grid: 16x30, 99 mines)
  • JSON (grid: 13x13, 8 mines)
